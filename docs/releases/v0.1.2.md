# Asterline v0.1.2

This release focuses on making native agent CLIs feel at home inside Asterline.

## Highlights

- Added targeted agent-skill completion and native command pass-through after
  prefixes such as `@codex /`.
- Preserved richer native CLI output, including structured tool-call results,
  reasoning, command output, and file-change details.
- Added persistent `/mode` selection with normal, review, plan, roundtable, and
  workflow modes.
- Added policy-driven approval gates for user dispatches, agent relays, and mode
  dispatches.
- Improved session continuity: fresh sessions are created once and pinned,
  explicit session IDs can be resumed, and `/team` now provides an internal
  Codex/Claude/Grok session table limited to the member's working directory.
- Reworked Team text editing with a focused input box, Unicode-safe cursor
  movement, shell-style editing shortcuts, atomic paste, and real cancel
  semantics.
- Improved terminal interaction: mouse text selection and clipboard copy,
  atomic bracketed paste, faster wheel scrolling, prompt history on arrow keys,
  bounded selection inside drawers, and `Esc` interruption behavior.
- Updated the team skill protocol so the visible roster only advertises
  available teammates and no longer implies automatic delegation.

## 中文更新摘要

- 支持在 `@codex /` 等成员前缀后补全并透传 Agent CLI 自带的 Skill 命令。
- 更完整地展示原生 CLI 工具调用、推理、命令输出和文件修改结果。
- 新增持久化 `/mode` 选择，包含正常、审查、规划、圆桌和工作流模式。
- 审批策略覆盖用户消息、Agent 中继和模式派发。
- 改进会话连续性，并在 `/team` 内提供仅限成员当前目录的会话选择表格。
- Team 字段使用独立编辑框，支持 Unicode 光标、Shell 风格快捷键和秒粘贴。
- 改进鼠标复制、滚轮滚动、上下历史消息、Drawer 选择边界和 `Esc` 打断。
- Team Skill 名单现在仅表示可用成员，不再默认触发不必要的 Agent 消息。
